# ROLE_EXECUTION_RULES_REPORT.md
v0.1

## Назначение

Отчёт фиксирует результат создания общего контракта исполнения ролей в `ROLE_EXECUTION_RULES.md`.

---

## Выявленные общие правила

В существующих ролевых инструкциях и канонических документах повторялись следующие общие правила:

- агент обязан прочитать обязательные входные артефакты перед началом работы;
- агент обязан проверить достаточность входных данных;
- агент обязан проверять противоречия между каноническими источниками;
- агент должен продолжать работу на разумных предположениях, если вопрос не влияет на бизнес-результат;
- агент должен готовить возврат, если проблема находится в результате предыдущего этапа или артефакте другой роли;
- агент должен готовить эскалацию только по основаниям из `DECISION_AUTHORITY.md`;
- агент не должен подменять соседние роли;
- агент должен создавать и обновлять артефакты по структуре из `ARTIFACTS.md`;
- агент должен фиксировать источники, причины, риски, ограничения, отклонения и последствия решений;
- результат роли должен быть трассируем к входным источникам;
- Project State является служебным состоянием Studio Director;
- Project Memory обновляется Historian и не должна изменяться остальными ролями напрямую.

---

## Что вынесено на общий уровень

В общий документ вынесены правила, которые не принадлежат одной конкретной роли:

- общий принцип ответственности агента за результат своей роли;
- порядок работы с входными данными;
- классификация недостаточности входов: предположение, возврат или эскалация;
- правила разумных предположений;
- правила обработки противоречий;
- правила создания и обновления артефактов;
- единый формат возврата;
- единый формат эскалации;
- правила использования Project State;
- правила взаимодействия с Project Memory;
- общая трассируемость результата;
- критерии качества работы роли;
- общие запреты для всех ролей.

Документ не переносит в общий уровень методики конкретных ролей: Discovery, Product Definition, Solution Design, Planning, Implementation, Validation, Release и Project Memory Update остаются в специализированных инструкциях.

## Обнаруженные противоречия

Критических противоречий, требующих изменения жизненного цикла, списка ролей или перечня канонических артефактов, не обнаружено.

Обнаружены две точки выравнивания:

1. `AGENTS.md` перечислял документы, за соблюдением которых следит Studio Director, но не мог упоминать новый `ROLE_EXECUTION_RULES.md`, потому что его ещё не существовало.
2. `DECISION_AUTHORITY.md` требовал сохранять самостоятельные решения в Project Memory. Для согласования с владением Project Memory уточнено, что сохранение происходит через артефакт, отчёт или процессное решение роли и последующее обновление Historian, а не прямой записью любой роли.
